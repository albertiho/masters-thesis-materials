# Statistical Tuning Results

This folder contains the compact outputs from the statistical detector tuning
runs. The full local runs also produced about 40.45 GiB of row-level parquet
artifacts, which are omitted from the normal GitHub checkout by `.gitignore`.

The omitted files are mainly:

```text
*/best_candidate/splits/new_prices/injected_rows.parquet
*/best_candidate/splits/new_prices/predictions.parquet
*/best_candidate/splits/new_products/injected_rows.parquet
*/best_candidate/splits/new_products/predictions.parquet
```

The omitted files are generated by:

```powershell
python research/training/scripts/tune_statistical.py
```

Use the concrete command arguments from the relevant `summary.md`,
`run_metadata.json`, or `research/training/scripts/README.md` section for the
run you want to reproduce.

For normal thesis replication, start with the included compact outputs:
`best_configuration.json`, `candidate_metrics.csv`, `scope_status.csv`, and the
aggregate analysis CSV files. Regenerate the parquet files when you need
per-row audit data or want to rebuild metrics from detector predictions. Full
statistical sweeps over the sampled minimum-history grid can take tens of
hours.
