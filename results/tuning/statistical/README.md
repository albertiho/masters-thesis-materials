# Statistical Tuning Results

This folder previously contained about 40.45 GiB of generated parquet files.
Those files are excluded from the GitHub release by `.gitignore`.

The omitted files are mainly:

```text
*/best_candidate/splits/new_prices/injected_rows.parquet
*/best_candidate/splits/new_prices/predictions.parquet
*/best_candidate/splits/new_products/injected_rows.parquet
*/best_candidate/splits/new_products/predictions.parquet
```

They are generated by:

```powershell
python research/training/scripts/tune_statistical.py
```

Use the concrete command arguments from the relevant `summary.md`,
`run_metadata.json`, or `research/training/scripts/README.md` section for the
run you want to reproduce.

The row-level parquet files are useful for audit and metric recomputation, but
the thesis figures usually consume compact outputs such as
`best_configuration.json`, `candidate_metrics.csv`, `scope_status.csv`, and the
aggregate analysis CSV files. Full statistical sweeps over the sampled
minimum-history grid can take tens of hours.
